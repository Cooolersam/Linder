Methodology

Overview

Linder measures how alike words for the same concept look and sound across languages. It compares 96 languages across 27 language families by scoring English-foreign translation pairs on orthographic and phonetic similarity, then extends this to cross-language comparison using English as a pivot.

Data Sources

Translation pairs come from three open datasets:

  • MUSE (Meta Research) — bilingual dictionaries for 44 languages, up to 100k+ pairs each. Space/tab-separated word pairs. CC-BY-NC license.
  • PanLex (Long Now Foundation) — lexical translations for 24 additional languages, extracted via shared meaning IDs from the HuggingFace cointegrated/panlex-meanings dataset. CC0 license.
  • FreeDict — dictd-format bilingual dictionaries for 6 languages (Welsh, Irish, Kurdish, Latin, Swahili, Urdu). GPL license.
  • Wiktionary (kaikki.org) — English Wiktionary JSONL dump, parsed for translation fields. 22 languages extracted. CC-BY-SA license.
  • Apertium — open-source MT bilingual dictionaries in XML format. Used to supplement Basque and Maltese. GPL license.
  • sinhala-para-dict — open-source English-Sinhala word pairs from GitHub. Used to supplement Sinhala (2k to 314k pairs).

Multi-word phrases are filtered out. Only single-word to single-word pairs are used. Pairs are deduplicated per language.

Romanization

Non-Latin scripts are converted to Latin characters before scoring. Each script uses a dedicated method:

  • Chinese — Pinyin via pypinyin. Tone marks stripped.
  • Japanese — Hepburn romanization via pykakasi.
  • Hindi, Bengali, Nepali, Marathi — IAST transliteration via indic-transliteration, then diacritical marks removed with Unicode NFD normalization.
  • Russian, Ukrainian, Bulgarian, Greek, Arabic, Hebrew, Korean, Georgian, Armenian, Thai, Khmer, Lao, Burmese, Mongolian, and others — unidecode, which provides approximate ASCII transliteration for any Unicode text.
  • Latin-script languages (French, German, Turkish, etc.) — diacritical marks stripped (e.g., "uber" from "uber") for fairer comparison.

Scoring Metrics

Each word pair is scored on 7 metrics, then combined into a weighted composite (0 to 1, where 1 is identical):

MetricWeightWhat it measures
Normalized Levenshtein30%Minimum character edits needed to transform one word into the other, normalized by length. Core orthographic measure.
Jaro-Winkler25%Character-matching similarity with a bonus for shared prefixes. Rewards words that start the same way.
Metaphone15%English phonetic encoding. Returns 1.0 if both words produce the same Metaphone code, 0.0 otherwise. Uses jellyfish.
Soundex10%Coarser phonetic encoding. Same binary scoring as Metaphone. Uses jellyfish.
Bigram Jaccard10%Overlap of character pairs (2-grams). Measures structural similarity at the substring level.
Trigram Jaccard5%Same as bigram but with character triples. Captures longer shared substrings.
Length Similarity5%1 minus the normalized length difference. Words of similar length score higher.

Metaphone and Soundex are English-centric phonetic algorithms. They contribute meaningful signal for European languages but less so for languages with very different phonologies. The composite score is still dominated by the script-agnostic metrics (Levenshtein, Jaro-Winkler, n-grams).

Cross-Language Comparison

To compare two non-English languages (e.g., French vs Japanese), English is used as a pivot:

  1. Find English words that have translations in both languages.
  2. Romanize both foreign translations.
  3. Score the romanized forms against each other using the same composite metric.

This measures whether two languages use similar-sounding words for the same concepts. The method correctly identifies known relationships: Bosnian-Croatian (0.77), Czech-Slovak (0.71), Indonesian-Malay (0.70), Spanish-Portuguese (0.69).

Limitations: the pivot approach only covers concepts that exist in all three languages' dictionaries. Scores may be biased toward loanwords that passed through English.

Network Graph

The force-directed network uses D3.js. Each language is a node, connected to its most similar neighbors (top 3) plus any pair scoring above 0.55.

  • Family placement — a preliminary force simulation runs on language families themselves, using inter-family similarity scores as edge weights. Similar families (e.g., Romance and Germanic) end up near each other.
  • Node sizing — radius scales with centrality (average similarity to all other languages) using a cubic power scale, so highly connected languages visually stand out.
  • Family gravity — each node is pulled toward its family's computed center. Nodes cannot drift beyond 120px from their family zone.
  • Inter-family repulsion — nodes from different families push each other apart when close, creating visible gaps between clusters.
  • Ambient drift — idle nodes float gently with per-node randomized drift vectors until the user interacts.

Compare Tool

The word compare feature uses pre-computed lookup dictionaries (one JSON file per language). If a word is not in the dictionary, it falls back to live translation via LibreTranslate free API mirrors. Client-side scoring in JavaScript reimplements Levenshtein, Jaro-Winkler, and n-gram Jaccard. Soundex/Metaphone are not available in JS, so their combined 25% weight is redistributed equally to Levenshtein and Jaro-Winkler.

Color Scheme

Node colors encode approximate geographic origin of each language family, following a west-to-east gradient:

  • Reds/oranges — Western/Northern Europe (Celtic, Germanic, Romance, Basque)
  • Rose/magenta — Southeastern Europe (Albanian, Hellenic)
  • Purples — Eastern Europe (Slavic, Baltic, Uralic)
  • Greens — Middle East / Western Asia (Semitic, Armenian, Kartvelian)
  • Olive — Central Asia (Turkic, Mongolic)
  • Teal — South Asia (Indo-Iranian, Dravidian)
  • Blues — East Asia (Sino-Tibetan, Japonic, Koreanic)
  • Cyan — Southeast Asia / Pacific (Tai-Kadai, Austroasiatic, Austronesian)
  • Browns — Africa (Niger-Congo, Chadic, Cushitic)
